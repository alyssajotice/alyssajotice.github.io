MIDI Sound Board

Spring 2017

One of the classes I took during freshman year was MUSIC 452, a class where I learned the programming language SuperCollider. SuperCollider is a language that "uses a servor for real-time audio synthesis and algorithm composition." One of my passions is music, and this was one of the classes I took to count towards my minor in Music Technology. Throughout the class, I learned about synthesis composition and the different types of waves that engineers use to create the sounds we hear in music today.

The class ended with a capstone project that could be about whatever we wanted. I decided to use a Korg Nantokontroll2 (pictured below) to control different aspects of synths that I had coded. There are nine different control sections on the board, so I decided to use nine different synths in the project. The top dial would control the length of the sound. This got a bit tricky because I had to be sure that shortening the sound wouldn't get rid of a characteristic part of the sound, such as the decay or release portion. The slider in the center of the board was programmed to change the volume of the sound, which wasn't that difficult. Lastly, the buttons to the left of the slider controlled which synth was selected, so each slider controlled a specific synth.


The synths I created for this project included as follows: red cedar wood plate, trumpet, reverb, bells, vibraphone, guitar, laser, vibrato, and flute. After creating all of these, I had to make sure they had the appropriate parameters so that their volume and length could be changed. Once all of those were complete, I had to get all of the MIDI data to work together.

The notes needed a way to be played, so I added a MIDI keyboard to the setup for more control. First, I set up the synths to work with the keyboard (pictured below). Just like a piano, the keys play different notes that come out at the correct frequency, but sound like the synths listed above.


The first thing I did was create global variables to hold the inputs on the sound board. The sound board data came in the form of MIDI signals, with a bus number for identification and a value in the range 0-127.

One of the biggest challenges I faced was getting the input data to line up with my code. I originally did the variable assignments by using subtraction and modular division for a range of busses, but I learned the hard way that certain bus numbers are reserved for things like master volume. To keep things clear and correct, I had to assign all of the values manually. I made a 9x2 array that held the current settings for each of the 9 synths controlled by the board. Then, when a button was pressed, that synth would be assigned to a global variable called "currentSynth". When a key on the keyboard was pressed, if statements were used to figure out which synth needed to be played.

SuperCollider can be very tempermental with the order in which things are initialized, because everything is run by a live server. The most frustrating part was having to shut down the servor and restart everything when I wanted to retest a change in my code. I ended up writing out the exact sequence of initializations for the global variables so that it worked properly each time. Overall, this project had some challenges that I had never faced before, but I learned a lot and ended up with a really cool end product.
